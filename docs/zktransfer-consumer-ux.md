# zkTransfer Consumer UX

## Product Principle

일반 사용자는 `zk`, `proof`, `note`, `witness`, `public/private amount`를 이해할 필요가 없다.

사용자에게 보여줄 모델은 아래 세 가지뿐이다.

- `빠른 송금`
- `개인정보 보호 송금`
- `자동 추천`

## Mental Model

사용자는 은행 송금처럼 아래 세 가지만 결정한다.

- 누구에게 보낼지
- 무엇을 보낼지
- 얼마를 보낼지

앱/서버가 자동으로 결정한다.

- 공개 송금으로 처리할지
- 개인정보 보호 송금으로 처리할지
- 혼합 처리할지
- 잔돈 note가 필요한지
- proving이 필요한지
- custody server가 어떤 경로로 tx를 만들지

## Recommended Labels

사용자에게 노출:

- `송금하기`
- `빠른 송금`
- `개인정보 보호 송금`
- `자동 추천`
- `요청 접수`
- `보안 검증`
- `전송 준비 중`
- `전송 완료`
- `영수증`

사용자에게 숨김:

- `zkTransfer`
- `proof`
- `witness`
- `note`
- `circuit`

## Transfer Flow

1. 사용자는 수신자, 자산, 금액을 입력한다.
2. UI는 기본적으로 `자동 추천`을 선택한다.
3. 시스템은 현재 입력과 자산 상태를 보고 추천 문구를 보여준다.
4. 사용자는 그대로 `송금 요청`을 누른다.
5. 서버는 proving/signing/submission을 처리한다.
6. 사용자는 상태 카드와 영수증만 본다.

## Recommendation Copy

- `자동 추천: 이 송금은 개인정보 보호가 유리합니다.`
- `자동 추천: 일반 송금으로 더 빠르게 처리할 수 있습니다.`
- `개인정보 보호 송금: 거래 노출을 줄이는 보호 모드입니다.`
- `빠른 송금: 일반 블록체인 송금으로 처리합니다.`

## Web vs App

공통:

- 같은 용어
- 같은 상태 모델
- 같은 송금 플로우

Web:

- 상태/활동/탐색을 넓게 보여준다.
- 운영 및 고급 사용자가 보기 좋게 설계한다.

App:

- 입력과 확인을 최소 단계로 줄인다.
- 한 손 사용과 빠른 반복 송금에 맞춘다.
